# AGENTS.md — инструкции для кодинг-агентов


Этот файл описывает, как AI-агент (Claude Code, Cursor, Copilot и др.)
должен работать с проектом лендинга **Mighty English**. Соблюдай эти правила
при любой генерации и правке кода.

---

## Что это за проект

Статический одностраничный лендинг авторской школы английского языка
Рудольфа Ладнера. Цель сайта — продать личный бренд преподавателя и привести
посетителя к записи через Telegram. Без бэкенда, без CMS, без базы данных.

---

## Обязательно прочитать перед работой

Перед любой задачей агент читает эти файлы и следует им:

1. **`DESIGN.md`** — дизайн-система (формат Google Stitch). Источник истины
   по цветам, типографике, отступам, компонентам. **Любой UI-компонент
   верстается строго по токенам из `DESIGN.md`.** Не выдумывать цвета,
   размеры и отступы.
2. **`mighty_english_landing_doc.md`** — полный рабочий документ. Источник
   истины по структуре секций (блок 3) и по всем текстам (блок 4 —
   копирайтер). **Тексты копировать оттуда дословно, не переписывать.**
3. **`README.md`** — обзор проекта, стек, структура, команды.

Если задача затрагивает внешний вид — `DESIGN.md` имеет приоритет.
Если задача затрагивает контент — `mighty_english_landing_doc.md` имеет
приоритет.

## Технологический стек

- **Astro** — фреймворк. Сайт статический, рендерится в HTML на сборке.
- **Нативный CSS** с custom properties. Токены из `DESIGN.md` лежат в
  `src/styles/tokens.css` как CSS-переменные.
- **Ванильный JavaScript** — по минимуму, только там, где без него нельзя.
- **Node.js 18+**.

Не вводить новые зависимости и фреймворки без явного запроса. Не добавлять
React/Vue/jQuery — проект на Astro и нативном CSS.

## Команды

```bash
npm install        # установка зависимостей
npm run dev        # дев-сервер, http://localhost:4321
npm run build      # сборка в ./dist
npm run preview    # предпросмотр сборки
```

проходит без ошибок.

## Правила вёрстки

### Дизайн-токены
- Все цвета, шрифты, отступы, скругления, тени — только из `DESIGN.md`
  через CSS-переменные. Хардкод значений (`#hex`, `16px`) в компонентах
  запрещён, кроме файла токенов.
- Имена переменных совпадают с токенами `DESIGN.md`: например
  `--color-primary`, `--color-accent`, `--rounded-md`, `--spacing-lg`.

### Структура
- Каждая секция лендинга — отдельный Astro-компонент в `src/components/`.
- Страница `src/pages/index.astro` собирает секции по порядку из
  `README.md` / блока 3 документа.
- Общий `<head>`, мета-теги, подключение шрифтов и Яндекс.Метрики —
  в `src/layouts/Base.astro`.

### Адаптивность
- **Mobile-first.** Базовые стили — для 375px, медиазапросы расширяют вверх.
- Брейкпоинты: 375 / 768 / 1024 / 1440 px.
- Проверять каждую секцию на всех брейкпоинтах.
- Размеры заголовков масштабировать через `clamp()` между мобильным и
  десктопным значениями из `DESIGN.md`.

### Доступность
- Семантический HTML: `<header>`, `<main>`, `<section>`, `<nav>`, `<footer>`.
- У всех изображений — осмысленный `alt` (тексты alt — в блоке 4.11
  документа).
- Контраст текста — не ниже WCAG AA. Палитра в `DESIGN.md` уже проверена.
- Уважать `prefers-reduced-motion` — отключать анимации.
- Все интерактивные элементы доступны с клавиатуры, видимый фокус.

### Производительность
- Изображения — в WebP/AVIF с фолбэком, через компонент `<Image>` Astro.
- Всё ниже Hero — `loading="lazy"`.
- Шрифты — `font-display: swap`, только нужные начертания.
- Видео — не нативное: YouTube-эмбед с фасадом (превью-картинка, плеер
  по клику).
- Ни одна анимация не должна опускать Lighthouse Performance ниже 90.
- Параллакс не использовать.

## Правила контента

- Тексты секций брать дословно из `mighty_english_landing_doc.md`,
  блок 4. Не сочинять и не перефразировать — голос Рудольфа уже выверен.
- Не менять цены, факты, цифры. Актуальные тарифы Мощного курса:
  Базовый 3 300 ₽, Расширенный 5 600 ₽, Премиум 9 600 ₽ (со скидкой 25%).
- Не выдумывать отзывы. Реальный отзыв — Диляра (выпускница 1-го потока);
  остальные предоставляет клиент.
- Все главные CTA ведут в Telegram-бот `https://t.me/MIGHTY_ENGLISH_info_bot`.
  Запасной контакт — `https://t.me/Mighty_Teacher`.

## CTA и ссылки

- Кнопки-ссылки в Telegram — обычные `<a href>` с `target="_blank"` и
  `rel="noopener"`. Никаких форм-прокладок.
- На каждый клик по CTA вешать событие Яндекс.Метрики (см. блок 5.3
  документа: цели `cta_bot_click`, `cta_teacher_click`, `cta_channel_click`).
- Ссылки из соцсетей размечать UTM-метками — не удалять их при обработке.

## SEO

- В `<head>` — `title`, `meta description`, Open Graph, Twitter Card.
  Готовые значения — в блоке 4.11 документа.
- Разметка Schema.org (JSON-LD): `EducationalOrganization`, `Person`,
  `Course`, `Review` — см. блок 5.4.
- `robots.txt` и `sitemap.xml` — в `public/`. Sitemap генерировать
  интеграцией Astro.
- Один `<h1>` на странице. Иерархия заголовков без пропусков уровней.

## Git и коммиты

- Коммиты атомарные, по одной логической задаче.
- Сообщения коммитов на русском или английском, в повелительном
  наклонении: «Добавить секцию Hero», «Исправить контраст кнопки».
- Не коммитить `node_modules/`, `dist/`, `.env`, локальные логи.
- Не коммитить служебные файлы редактора.

## Чего НЕ делать

- Не менять дизайн-токены `DESIGN.md` без явного запроса. Если правка
  нужна — сначала обновить `DESIGN.md`, потом код.
- Не добавлять бэкенд, базу данных, CMS — проект статический.
- Не вводить новые npm-зависимости без необходимости и согласования.
- Не использовать `localStorage`/`sessionStorage` и браузерные хранилища
  без явной задачи.
- Не переписывать тексты Рудольфа «под себя».
- Не использовать стоковые фото людей — только материалы из `public/images`.
- Не ставить чисто-белый фон страницы — фон только тёплый off-white
  (`--color-surface`), см. `DESIGN.md`, секция Do's and Don'ts.
- Не вставлять корпоративные шаблоны и не терять живой, рукотворный
  характер бренда.

## Definition of Done

Задача считается выполненной, когда:

- [ ] Код собирается: `npm run build` проходит без ошибок.
- [ ] Вёрстка соответствует токенам `DESIGN.md`, хардкода значений нет.
- [ ] Секция корректна на 375 / 768 / 1024 / 1440 px.
- [ ] Тексты совпадают с `mighty_english_landing_doc.md`, блок 4.
- [ ] Семантический HTML, у изображений есть `alt`, контраст AA соблюдён.
- [ ] CTA ведут в нужный Telegram, события Метрики навешаны.
- [ ] Lighthouse (mobile): Performance ≥ 90, прочие метрики ≥ 95.
